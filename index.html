如果 x == y，那么两块石头都会被完全粉碎；

如果 x != y，那么重量为 x 的石头将会完全粉碎，而重量为 y 的石头新重量为 y-x。

最后，最多只会剩下一块石头。返回此石头最小的可能重量。如果没有石头剩下，就返回 0。

组合 2 和 1，得到 1，所以数组转化为 [1,1,1]，
组合 1 和 1，得到 0，所以数组转化为 [1]，这就是最优值。

public:
    TreeNode* compare(vector<int>& nums, int a, int b) {
        if (a > b) return NULL;

        // 找到 [a, b] 范围内的最大值索引
        int index = a;
        for (int i = a; i <= b; i++) {
            if (nums[i] > nums[index]) {
                index = i;
            }
        }

        // 创建根节点
        TreeNode* root = new TreeNode(nums[index]);

        // 递归构建左子树和右子树
        root->left = compare(nums, a, index - 1);
        root->right = compare(nums, index + 1, b);

        return root;
    }

    TreeNode* constructMaximumBinaryTree(vector<int>& nums) {
        return compare(nums, 0, nums.size() - 1);

又一个组合总和

给定一个无重复元素的数组 candidates 和一个目标数 target ，找出 candidates 中所有可以使数字和为 target 的组合。 candidates 中的数字可以无限制重复被选取。
所有数字（包括 target）都是正整数。
解集不能包含重复的组合。
示例 1：
输入：candidates = [2,3,6,7], target = 7,
所求解集为： [ [7], [2,2,3] ]
示例 2：
输入：candidates = [2,3,5], target = 8,
所求解集为： [ [2,2,2,2], [2,3,3], [3,5] ]

这种给定一个限制范围 之后多次在这一个集合范围里面组合的问题需要注意for循环的起始点
这次的终止条件就是求和大于等于要求的和了 如果没有达到可以无限循环下去
同时因为重复选取所以递归限制解除

组合总和终：这个给定集合有重复元素 但是提供的结果还不能有重复组合

给定一个数组 candidates 和一个目标数 target ，找出 candidates 中所有可以使数字和为 target 的组合。candidates 中的每个数字在每个组合中只能使用一次。
说明： 所有数字（包括目标数）都是正整数。解集不能包含重复的组合。

明白这种退回的思想 让path 在添加同时也能回溯 用一个固定的集合去到达目标

这道题不同点在于不可重复 同时给的区间里面还有重复 要在搜索的过程中就去掉重复组合

对于这种类型 因为给定的集合本身可以重复所以在纵向树枝上可以正常递归的时候重复 同时在同一层进行for循环的时候为了避免之前在纵向递归的时候与其重复，所以在横向遍历的时候去重操作

需要首先在横向树层进行排序 去重本身含义可以理解已经使用过的元素不能再用了

注意最后需要排序 这样才能有效把相同的树聚在一起
